# AegisOS v31.1 — Controlled EL0 Transition Tree Guard

v31.1 does not advance the milestone beyond v31.0. It keeps the controlled EL0 transition scope and adds a tree guard so stale folders are caught before running QEMU.

Why this exists:

- A run from an old tree can still build and pass v29, which looks like an EL0 failure but is not.
- v31 requires `boot/arm64/el0_transition.S`, the v30 pre-EL0 checkpoints, and the v31 controlled EL0 checkpoints.
- `tools/qemu/build-and-smoke-aarch64.sh` now calls `tools/dev/assert-v31-line.sh` before building.

Expected v31 success:

```text
QEMU v31.0 proof accepted: controlled EL0 eret transition entered EL0 stub and trapped back through SVC.
```

If the guard fails, clean replace the tree:

```bash
cd ~
rm -rf AegisOS
unzip ~/Downloads/AegisOS-router-bringup-v31.1-controlled-el0-transition-treeguard.zip
cd ~/AegisOS
tools/dev/fix-clock-skew.sh
tools/qemu/build-and-smoke-aarch64.sh
```
